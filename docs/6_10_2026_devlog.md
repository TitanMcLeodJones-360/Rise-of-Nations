## Topic: Variables, Undefined Behavior, and Coding Conventions

Today I learned about initialized and uninitialized variables.

An initialized variable is a variable that has been given a value when it is created.

```cpp
int population{1000};
```

An uninitialized variable is a variable that has not been given a value.

```cpp
int population;
```

Using an uninitialized variable can lead to **undefined behavior**. Undefined behavior occurs when code executes in a way that is not well-defined by the C++ language. The result can be unpredictable and may vary between different systems, compilers, or program executions.

Learning this reinforced the importance of properly initializing variables before using them in a program.

### Application to Rise of Nations

This concept will be important when managing game data such as:

* Population
* Oil reserves
* Metal reserves
* Food production
* Military strength
* National treasury

If these variables are not initialized correctly, the game could produce incorrect values or unexpected behavior.

### Identifier Naming Conventions

I also learned about common naming conventions for identifiers.

Good variable names should:

* Start with a lowercase letter
* Be descriptive
* Explain the purpose of the variable
* Avoid whitespace
* Not begin with a number

Examples:

```cpp
int populationCount;
int militaryStrength;
int oilReserves;
```

Poor examples:

```cpp
int 123population;
int my variable;
int x;
```

Following consistent naming conventions makes code easier to read and maintain.
